# 不完美作战 v5.0 — 核心机制策划案

**版本**: v5.0 完全重写  
**日期**: 2026-05-08  
**来源**: 用户策划案 + 工程细化

---

## 一、游戏基础框架与视觉交互

### 1.1 进入流程
- 主界面：难度 n1~n10 梯队选择
- 角色选择：两大体系（无UI体系 / 带UI体系）
- 选定后无缝载入战斗地图

### 1.2 视角
- 俯视视角，镜头严格锁定人物居中
- 怪物默认从屏幕上端口涌出，向下施加压迫

### 1.3 美术规范
- 1:1 比例图鉴风格（1:1 scale illustrated catalog style）
- 扁平大色块，磨砂玩具质感
- 物理交互反馈：清脆拼装声、碎裂纯色特效
- 去除多余光影
- **卡牌不透明限制已移除**——界面轻盈通透

## 二、局内宏观节奏与双轨经济

### 2.1 标准局时长
- **30 分钟**，精准切分 **5 大关**
- 每大关包含 **4 小波次**（含精英/高压节点）

### 2.2 高压无休整
- 局内**无商店休整期**
- 所有构筑在战斗中完成

### 2.3 双轨经济
| 资源 | 用途 |
|------|------|
| 金币 | 抽卡运转 |
| 杀敌数 | 购买全局天赋 |

---

## 三、终极奖励体系（道具与掉落）

### 3.1 5 格极限容量
- 全局快捷道具栏绝对上限 **5 格**
- Boss/精英掉落后，有空位则自动存入
- 满载时，道具实体掉落在英雄脚下（物理空间）

### 3.2 主动丢弃
- 无快捷键，必须直接点击 UI 释放
- 支持拖拽/长按主动丢弃
- 高压下可扔掉次要道具拾取关键道具

### 3.3 跟随型 Boss 道具
- Boss 掉落道具/武器跟随英雄悬浮移动
- 玩家必须点击使用触发效果

### 3.4 特权出怪口（资源管道）
- Boss 掉落"超级阀门/胶囊管道"
- 点击使用出现"选择出怪品类"界面
- 可选：掉落永久属性的【沙袋暴徒】、掉落海量金币的【纯金史莱姆】等
- 可收起界面延期选择

## 四、全局天赋系统

### 4.1 触发方式
- 消耗【杀敌数】
- 右下角常驻"残骸储能管"，满能后弹出 3 个磨砂塑料质感实体卡带

### 4.2 交互
- 点击卡带→"咔哒"音效→飞入【全局天赋图鉴 UI】常驻亮起
- 选中瞬间，其余 2 卡带碎裂成像素沙粒消失（无延迟）

### 4.3 机制示例
- 【自动贩卖机】：每 30 秒强塞 1 张打工卡
- 【贪婪胶囊】：收益翻倍，但深渊盲盒必出 2 张诅咒卡

---

## 五、核心 6 格悬浮卡槽体系

### 5.1 悬浮 6 格
- 主界面 6 卡槽悬浮设计
- 触碰/悬停时卡牌瞬间放大 + 弹出极简简介

### 5.2 溢出替换
- 卡槽满载后新卡进入"溢出替换交互"
- 必须强行顶替一张旧卡

### 5.3 出售限制
- 出售按钮**仅在 6 格卡牌简介弹窗中**
- 防误触

### 5.4 刮刮乐回收
- 顶替/摧毁卡牌时按阶级触发返现
- 概率爆卡 + 纯色碎裂特效

---

## 六、两大核心英雄体系

### 6.1 无 UI 体系（精准对子流）
- 纯靠 6 格卡槽配对公式运转
- 示例：【寿司忍者】
- 终极羁绊奖励：召唤专属悬浮火力宠物（如三文鱼锦鲤飞艇）

### 6.2 带 UI 体系（修仙/藏剑流派）
- 专属 UI 悬浮跟随人物移动
- 点击进入二级 UI（含回收 + 升级专属武器入口）

#### L1→L2→L3 回收演进
```
L1 基础打工卡 → 打怪积攒"回收次数"
L2 选择（多选一，选定后其余流派 L2 不再出现）
    → 10秒生命周期 → 自动判定成功率 → 回收
L3 产出
    → 普通 L3（辅助型）：提高回收效率/成功率/触发概率
    → 高级 L3（核心与养成）：提供技能和伤害
    → 回收成功后 +1 升级专属武器次数（含成功率判定）
```

#### 大招共鸣
- 集齐 6 张特定高级羁绊卡
- 触发终极大招，叠加在技能体系之上

## 七、极简战术 HUD

### 7.1 英雄蓝图
- 点击右上角图标滑出半透明侧边数据栏
- 显示：【核心组件】→【羁绊回路】→【外挂模块（天赋）】

- **绝对不包含售卖按钮**
- 查阅空间与操作空间彻底隔离

## 八、技术架构规划（v5）

### 8.1 模块拆分
```
public/modules/
├── engine-core.js       # Canvas/渲染/相机/坐标
├── engine-input.js      # 点击/触摸/拖拽/悬停
├── engine-audio.js      # 音效（机械反馈音效）
├── game-data.js         # 配置（英雄/卡牌/天赋/难度）
├── game-state.js        # 全局状态管理
├── game-loop.js         # 主循环
├── game-combat.js       # 战斗/弹道/怪物AI
├── game-waves.js        # 波次/关卡/精英/Boss
├── game-cards.js        # 6格悬浮卡槽/溢出替换/回收
├── game-items.js        # 5格道具栏/掉落/跟随Boss道具
├── game-talents.js      # 天赋系统/残骸储能管/图鉴
├── game-heroes.js       # 两套英雄体系（无UI/带UI）
├── game-cultivation.js  # 修仙/藏剑 L1-L3 回收体系
├── game-render.js       # 渲染层
├── game-ui.js           # HUD/英雄蓝图/弹窗
└── game-main.js         # 入口/初始化
```

### 8.2 依赖关系
```
game-data → game-state → engine-core → engine-input → engine-audio
        → game-loop → {combat, waves, cards, items, talents, heroes, cultivation}
        → game-render → game-ui → game-main
```
