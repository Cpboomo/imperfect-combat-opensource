## 1. 游戏概述

**类型**: 俯视角 Roguelite 动作生存

**一句话**: 选择英雄、击败波次怪物、拾取装备构筑 Build、爬塔变强。

**平台**: 移动端 H5（微信/抖音/浏览器直接打开）

**核心循环**:
```
战斗(杀怪) → 金币/装备掉落 → 波间商店强化 → 更强怪物 → 卡牌选择 → 继续战斗
```

## 2. 设计支柱

| 支柱 | 描述 |
|------|------|
| **快速上手** | 点触移动、自动攻击，零学习成本 |
| **Build 构筑** | 装备+卡牌+修仙三系统叠加，每局不同体验 |
| **爽快战斗** | 粒子特效、弹道飞行、击杀反馈、范围清屏 |
| **可复玩性** | 随机掉落、多英雄、难度梯度、修仙分支 |

## 3. 英雄系统

| 英雄 | 定位 | 特点 |
|------|------|------|
| 🧙 修仙大仙 | 远程法系 | 190射程、飞剑追击(30%额外弹道)、高MP、修仙专属系统 |
| 🗡️ 忍术忍者 | 近战刺客 | 140射程、暴击+15%、闪现200距离、高攻击 |
| 🔮 超能力者 | 念力控场 | 220最远射程、大范围弹道、高MP回复、念力风暴 |

## 5. 技术架构

```
public/
├── game.html    — 游戏主页面 (638行)
├── game.js      — 游戏全部逻辑 (3316行)
├── editor.html  — 关卡编辑器
└── editor.js    — 编辑器逻辑

server.js        — Node.js后端 (加载/保存/版本管理)
```

**渲染**: 2D Canvas，360×640 基础分辨率，自适配缩放
**音效**: Web Audio API 合成音效（无音频文件依赖）
**寻路**: A* 网格寻路（18×32 网格，40px 格子）
**移动**: Lerp 速度跟随 + 惯性滑行（v4.3.7）
