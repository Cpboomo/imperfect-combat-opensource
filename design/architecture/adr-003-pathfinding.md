# ADR-003: 寻路方案 — A* 网格寻路

**状态**: Accepted  
**日期**: 项目初期  
**决策者**: 开发者

## 背景

点击移动需要找到从玩家位置到目标位置的最短路径，绕过墙壁障碍物。

## 决定

**使用 A* 算法在 18×32 网格上进行寻路。**

- 网格尺寸: 40px
- 地图尺寸: 720×1280px (18×32 格)
- 启发函数: 曼哈顿距离
- 移动代价: 均一（无地形差异）

## 理由

| 因素 | 分析 |
|------|------|
| 实现简单 | A* 是经典算法，~50 行代码 |
| 性能足够 | 18×32 = 576 节点，A* 在 1ms 内完成 |
| 地图结构 | 墙壁正好是格子化的，天然适配网格寻路 |
| 路径平滑 | 路径点用于导航线绘制（虚线引导） |

## 后果

- ✅ 点击移动响应即时
- ✅ 路径可视化（虚线引导）提升 UX
- ❌ 网格精度限制（40px 步长，无法斜角精确移动）
- ❌ 动态障碍物（怪物）不参与寻路

## 移动演进

- v4.3.5: 引入阻尼物理（→v += m(→F_推+→F_阻)·dt）
- v4.3.7: 阻尼不适配点触 → 改为 Lerp 速度跟随 + 惯性滑行

## 替代方案

| 方案 | 为什么不选 |
|------|-----------|
| NavMesh | 没有 3D/复杂地形，过度工程 |
| Dijkstra | 地图小，A* 启发式更快 |
| 射线检测直接移动 | 无法绕过 U 型墙 |
