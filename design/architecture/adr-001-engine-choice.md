# ADR-001: 引擎选型 — 纯 HTML5 Canvas

**状态**: Accepted  
**日期**: 项目初期  
**决策者**: 开发者

## 背景

需要选择一个游戏引擎来实现"不完美作战"。需要考虑的因素：零安装门槛、移动端兼容、快速迭代。

## 决定

**选择纯 HTML5 Canvas + Vanilla JavaScript，不依赖任何框架。**

不使用 Godot/Unity/Unreal 或 Phaser/Pixi.js。

## 理由

| 因素 | 分析 |
|------|------|
| 零安装 | 微信/抖音/浏览器直接打开，无需下载 |
| 开发速度 | 无框架学习成本，直接写 JS |
| 性能 | 2D Canvas 在移动端足够，WebGL 不需要 |
| 包体积 | 单个 HTML + JS 文件，<200KB |
| 分发 | GitHub Pages 一键部署 |

## 后果

- ✅ 极低分发门槛
- ✅ 快速迭代（改文件即上线）
- ❌ 无引擎提供的碰撞/物理/粒子系统
- ❌ 需手写 A* 寻路、弹道系统
- ❌ 无法利用 Godot/Unity 的编辑器工具

## 替代方案

| 方案 | 为什么不选 |
|------|-----------|
| Godot 4 | 导出 H5 体积大(>10MB)，微信不支持 |
| Unity | WebGL 构建慢，移动端性能差 |
| Phaser.js | 增加依赖，学习成本，Canvas API 更直接 |
